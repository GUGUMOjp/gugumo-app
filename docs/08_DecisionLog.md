# GUGUMO Architecture Decision Records (ADR)

Version: 1.0

Status: Active

---

# ADR-001 目的

本ドキュメントは、GUGUMOにおける重要な設計判断を記録する。

目的は以下の通り。

- 設計判断の理由を残す
- 将来の設計変更時の判断材料とする
- 新規開発者が背景を理解できるようにする
- 同じ議論を繰り返さない

ADRはソースコードより長く残る資産である。

---

# ADR-002 記録ルール

以下の場合はADRを追加する。

- アーキテクチャ変更
- データ構造変更
- 技術選定
- 開発方針変更
- ビジネスロジック変更
- 全国展開に影響する変更

軽微な修正は対象外。

# ADR-004

## タイトル

Rule Engineをサーバー側へ配置する

状態

採用

背景

GUGUMOの競争優位は判定ロジックにある。

決定

採用理由

ノウハウ保護

保守性

API化しやすい

全国展開しやすい

採用しなかった案

Frontend判定

理由

ロジック漏洩リスクが高い。
